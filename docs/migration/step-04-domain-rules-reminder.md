# Step 04 规则迁移文档：保养提醒规则

## 1. 目标与范围
- 目标：迁移提醒计算规则，保证 Android 与 iOS 行为一致。
- 范围：进度计算、颜色分段、排序、无记录场景。
- 非范围：页面 UI 细节。

## 2. 前置依赖
- 主约束：`step-01-foundation-tech-decisions.md`
- 上一步：`step-03-data-model-room-schema.md`

## 3. 设计与实现清单
- 核心规则
  - 里程进度 = 已使用里程 / 里程间隔。
  - 时间进度 = 已过时间 / 月份间隔总时间。
  - 时间与里程同时存在时：取进度更高者（谁先到采用谁）。
- 基线来源
  - 有历史记录：取项目最近一次记录的日期与里程。
  - 无历史记录：日期基线用购车日期，里程基线用 0。
- 展示与排序
  - 进度百分比保留 iOS 同口径。
  - 颜色分段：normal / warning / danger。
  - 排序：先按进度降序，再按优先级，再按项目名。
- 边界条件
  - 间隔为 0 或未启用提醒时处理策略。
  - 进度小于 0 与超出 100% 的展示策略。

## 4. 验收标准
- 典型样本输入输出与 iOS 对齐。
- 规则函数纯函数化，可单测。
- 完成定义（DoD）
  - 规则输入/输出模型定义明确。
  - 边界条件覆盖列表完整。
  - 排序规则与颜色规则可验证。

## 5. 风险与回滚
- 风险
  - 时区与日期归一化不一致导致天数计算偏差。
  - 提醒阈值迁移不一致导致颜色误判。
- 回滚策略
  - 以 iOS 规则快照样本为金标准，发现偏差先回退规则实现。
