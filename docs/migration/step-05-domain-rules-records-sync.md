# Step 05 规则迁移文档：保养记录保存与同步规则

## 1. 目标与范围
- 目标：迁移记录新增/编辑流程中的一致性规则。
- 范围：去重、关系重建、周期键同步、重复周期拦截。
- 非范围：记录页 UI 表现。

## 2. 前置依赖
- 主约束：`step-01-foundation-tech-decisions.md`
- 上一步：`step-04-domain-rules-reminder.md`

## 3. 设计与实现清单
- 周期键规则
  - `cycleKey = carID + yyyy-MM-dd(startOfDay)`。
  - 保存前统一归一化到日期维度。
- 周期项目键规则
  - `cycleItemKey = cycleKey + itemID`。
  - 保存时重建 `MaintenanceRecordItem` 关系。
- 项目去重
  - `itemIDsRaw` 保存前去重并保序。
- 重复周期处理
  - 新增/编辑均校验同车同日冲突，命中则返回业务错误。
- 联动更新
  - 记录保存后，车辆里程按“取较大值”更新，防止里程回退。

## 4. 验收标准
- 重复保存场景不会产生重复周期记录或重复周期项目关系。
- 关系表与主记录一致，不出现孤儿数据。
- 完成定义（DoD）
  - `cycleKey` / `cycleItemKey` 计算与回写规则明确。
  - 保存、编辑、冲突处理流程可验证。
  - 里程联动更新规则明确。

## 5. 风险与回滚
- 风险
  - 编辑场景下关系重建不完整造成数据不一致。
  - 并发写入导致唯一键冲突异常未被正确转换为业务错误。
- 回滚策略
  - 关键写路径采用事务；异常统一映射后再上抛。
