# Step 07 兼容文档：JSON 备份恢复与 iOS 完全对齐

## 1. 目标与范围
- 目标：实现与 iOS `MyDataTransferPayload` 双向完全兼容。
- 范围：JSON 结构、字段语义、导入校验、报错口径、导出约束。
- 非范围：文件选择器 UI。

## 2. 前置依赖
- 主约束：`step-01-foundation-tech-decisions.md`
- 上一步：`step-06-domain-rules-car-management.md`

## 3. 设计与实现清单
- 数据契约
  - 根节点：`modelProfiles`、`vehicles`。
  - 日期字段：`yyyy-MM-dd`，仅日期不含时区。
- 导入流程
  - 先校验 payload，再清空业务数据，再按顺序写入配置/车辆/记录。
  - 项目映射：优先 `catalogKey`，再按名称兼容。
- 校验规则
  - 车型配置重复、车辆 ID 重复、记录 ID 重复、日期格式非法、空项目等。
- 导出流程
  - 输出结构与字段命名与 iOS 一致。
  - 保持确定性排序，减少跨端 diff 噪声。

## 4. 验收标准
- 使用 iOS 导出样本可在 Android 成功导入。
- Android 导出样本可被 iOS 侧脚本/逻辑正确识别。
- 完成定义（DoD）
  - 契约字段与校验规则文档化完成。
  - 导入异常列表与错误文案策略完成。
  - 双向兼容回归样本清单完成。

## 5. 风险与回滚
- 风险
  - 名称映射与 key 映射优先级错误导致项目错配。
  - 日期解析宽松导致脏数据写入。
- 回滚策略
  - 导入使用严格校验与事务；失败即整体回滚。

## 6. 决策记录
- 本步骤无新增选型。
- 沿用 Step 01 既定决策：JSON 完全兼容 iOS，不引入新格式。
