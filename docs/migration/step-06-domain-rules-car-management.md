# Step 06 规则迁移文档：车辆管理与隔离规则

## 1. 目标与范围
- 目标：迁移车辆管理相关规则，保证数据隔离与回退行为一致。
- 范围：新增/编辑/删除、应用车型、禁用项目隔离、车型唯一性口径。
- 非范围：车辆表单 UI 细节。

## 2. 前置依赖
- 主约束：`step-01-foundation-tech-decisions.md`
- 上一步：`step-05-domain-rules-records-sync.md`

## 3. 设计与实现清单
- 车型唯一口径
  - 同一 `(brand, modelName)` 仅允许一辆车。
- 应用车型规则
  - 存储 `applied_car_id`，失效时回退到第一辆车或空。
- 删除车辆规则
  - 级联删除该车记录。
  - 删除 `ownerCarID` 命中的项目配置。
  - 若删的是当前应用车型，触发应用车型回退。
- 禁用项目隔离
  - 通过 `disabledItemIDsRaw` 对当前车生效，不影响其他车。

## 4. 验收标准
- 应用车型在新增/删除/恢复后始终可解析到有效值或空值。
- 跨车数据不会串扰。
- 完成定义（DoD）
  - 车型唯一校验规则明确。
  - 删除联动规则明确。
  - 应用车型回退规则明确。

## 5. 风险与回滚
- 风险
  - 应用车型失效时若无回退可能导致空白页或崩溃。
  - 删除联动缺失导致孤儿项目或记录。
- 回滚策略
  - 对删除流程使用事务与后置校验；出现异常时回滚。

## 6. 决策记录
- 本步骤无新增选型。
- 沿用 Step 01 约束：`ownerCarID` 隔离不可破坏。
