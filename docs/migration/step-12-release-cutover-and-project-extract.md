# Step 12 收尾文档：发布切换、目录迁出与独立项目化

## 1. 目标与范围
- 目标：定义迁移收尾标准，并把 `car-record-android` 平滑迁出为独立项目。
- 范围：完成定义、迁出步骤、风险控制、后续 backlog。
- 非范围：新功能开发。

## 2. 前置依赖
- 主约束：`step-01-foundation-tech-decisions.md`
- 上一步：`step-11-regression-test-and-acceptance.md`

## 3. 设计与实现清单
- 迁移完成定义（全项目）
  - 数据层、规则层、JSON 兼容、上下文层、UI 基础壳均已达标。
  - 回归测试与一致性验收通过。
- 目录迁出步骤
  - 冻结当前迁移分支。
  - 导出 `car-record-android` 到新仓库。
  - 重建新仓库 CI、版本号、签名与发布配置。
  - 补齐 README、架构文档、迁移历史文档。
- 风险控制
  - 保留迁移期间映射表（iOS -> Android 规则对应关系）。
  - 对已知差异建立 backlog 与优先级。

## 4. 验收标准
- 迁出后项目可独立构建、测试、运行。
- 关键文档（Step 01-12）完整保留。
- 完成定义（DoD）
  - 迁出操作清单可执行。
  - 发布前核对项完整。
  - 已知差异与后续计划已登记。

## 5. 风险与回滚
- 风险
  - 迁出时遗漏配置或脚本，导致新仓库不可用。
  - 文档与实际实现偏离，后续维护困难。
- 回滚策略
  - 迁出前打标签；迁出失败可回到标签点重试。

## 6. 决策记录
- 本步骤无新增技术选型。
- 迁出流程如需新增工具链，必须先补候选对比并更新 Step 01。
