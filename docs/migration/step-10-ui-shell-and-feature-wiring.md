# Step 10 UI 文档：三 Tab 壳与功能接线

## 1. 目标与范围
- 目标：建立 Android 端与 iOS 对齐的基础信息架构与入口接线。
- 范围：三 Tab 结构、页面入口、ViewModel 绑定、基础导航。
- 非范围：视觉细节打磨与高保真交互。

## 2. 前置依赖
- 主约束：`step-01-foundation-tech-decisions.md`
- 上一步：`step-09-data-layer-integration-and-repository.md`

## 3. 设计与实现清单
- 根结构
  - Tab：`保养提醒`、`保养记录`、`个人中心`。
- 接线原则
  - 每页只通过 ViewModel 调用 repository。
  - 导航跳转通过 NavigationRequestContext/导航层统一处理。
- 最小交付
  - 三个入口页可展示核心状态。
  - 新增记录入口与个人中心数据管理入口可联通数据层。

## 4. 验收标准
- 三 Tab 壳可运行，基础链路可通。
- ViewModel 与 repository 链路稳定。
- 完成定义（DoD）
  - 三入口与关键跳转路径定义完成。
  - 接线边界（UI -> VM -> Repo）符合约束。

## 5. 风险与回滚
- 风险
  - 过早追求视觉细节影响规则迁移稳定性。
  - 导航状态和上下文状态双源导致不一致。
- 回滚策略
  - 先保证“行为一致”，视觉问题后置优化。

- 若出现导航技术栈新增选项，必须先补“候选对比+影响面”并回写 Step 01。
